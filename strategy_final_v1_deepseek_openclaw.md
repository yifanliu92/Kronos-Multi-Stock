### 核心问题定位

> 方向预测接近随机（48%），但持仓 P&L 胜率仅 10.5%——不是方向判断太差，而是**开仓后没有主动退出机制，浮亏硬扛直到反转**。

持仓不变时的胜率为 **0.0%**（51 次全部浮亏），说明一旦开仓后方向不对，策略缺乏任何主动离场的手段。

## 二、优化方案（5 项参数调整）

### 设计原则

- 不改变信号层（方向预测仍为 48%）
- 不改变执行频率（仍然每 10 分钟一次）
- 不改变仓位上限规则（满仓后不追加新资金）
- 不改脚本、不改 cron、不新增文件
- 仅修改 `simulate_rules_603305.json` 的 5 个参数区块

### 改动一览

| # | 参数区块 | 当前值 | 目标值 | 设计思路 | 数据依据 |
|---|----------|--------|--------|----------|----------|
| 1 | **时间止损** | 无 | 连续 3 个时隙无反转信号 → 强制减仓至 20% | 避免开仓后死扛 | 持仓不变胜率 0% |
| 2 | **早盘禁仓** | 全天可加仓 | 09:30-10:00 禁止加仓，仅允许减仓/平仓 | 回避开盘噪声期 | 早盘胜率 0-5.6% |
| 3 | **止盈降级** | 3%/5%/7% | **1.2%/2.0%/3.0%** | 降低止盈门槛，让更多次小盈利触发锁利 | 影子策略已跑通，81%时点优于主策略 |
| 4 | **回撤保本** | 无 | 盘中浮盈后回撤 0.3% → 减仓 50% | 保护来之不易的小幅浮盈 | 83% 亏损从浮盈转亏开始 |
| 5 | **下午加成** | 无 | 13:00-14:00 加仓幅度 +10%/+15% | 仅在唯一有效的盈利窗口加重暴露 | 该时段胜率 32.1%，其余时段 0-5.6% |

### 详细参数计划

#### 改动 1：新增时间止损

在 `simulate_rules_603305.json` 中新增：

```json
"time_based_stop": {
  "enabled": true,
  "max_slots_without_signal_change": 3,
  "reduce_to_pct": 20,
  "exclude_hours": ["13", "14"]
}
```

**触发逻辑：** 连续 3 次检查（30 分钟）信号方向未变化 → 强制减仓至 20%。

**排除下午** 13:00-14:00 因为该时段是唯一盈利窗口，不应仓促离场。

#### 改动 2：早盘禁仓

在 `position_management` 中新增：

```json
"morning_cool_down": {
  "enabled": true,
  "start": "0930",
  "end": "1000",
  "actions_allowed": ["reduce_only", "flat"]
}
```

**09:30-10:00 期间**：所有加仓/加空动作被禁止，仅允许减仓和平仓。

#### 改动 3：止盈降级

`take_profit.profit_levels` 修改：

```json
// 当前
{ "profit_pct": 3.0, "reduce_pct": 30 },
{ "profit_pct": 5.0, "reduce_pct": 30 },
{ "profit_pct": 7.0, "reduce_pct": 20 }

// 改为（参考影子策略 v1.1-shadow）
{ "profit_pct": 1.2, "reduce_pct": 30 },
{ "profit_pct": 2.0, "reduce_pct": 15 },
{ "profit_pct": 3.0, "reduce_pct": 20 }
```

#### 改动 4：回撤保本

`take_profit.drawdown_levels` 增补一条：

```json
// 当前
{ "drawdown_pct": 1.0, "reduce_pct": 30 },
{ "drawdown_pct": 1.8, "reduce_pct": 30 },
{ "drawdown_pct": 2.5, "reduce_to_floor_pct": 20 }

// 改为：头部插入一条保本回撤减仓
{ "drawdown_pct": 0.3, "reduce_pct": 50 },  // 新增
{ "drawdown_pct": 1.0, "reduce_pct": 30 },
{ "drawdown_pct": 1.8, "reduce_pct": 30 },
{ "drawdown_pct": 2.5, "reduce_to_floor_pct": 20 }
```

**优先级：** 回撤止盈 > 时间止损 > 浮盈止盈 > 信号调仓。0.3% 回撤减仓最先触发，浮盈止盈 1.2% 其次。

#### 改动 5：下午加成

在 `long_side_actions` 和 `short_side_actions` 基础上，新增下午窗口参数：

```json
"afternoon_enhance_window": {
  "enabled": true,
  "start": "1300",
  "end": "1400",
  "bull_add_pct_trade": 10,
  "strong_bull_add_pct_trade": 15,
  "bear_add_pct_trade": 10,
  "strong_bear_add_pct_trade": 15
}
```

## 三、评审过程记录

### 第一轮（DeepSeek 提 → OpenClaw 评）

| DeepSeek 原始提案 | OpenClaw 评审 |
|-------------------|---------------|
| 120 秒计时器 | 🔴 执行分辨率不匹配，Kronos 是 10 分钟线 |
| 移动止盈 | 🟢 思想采纳，适配为参数级改动 |
| 固定 120s 条件 | 🔴 96% 仓位会因时间耗尽而非条件触发平仓 |

### 第二轮（DeepSeek 回应 → OpenClaw 再审）

| DeepSeek 新增建议 | OpenClaw 结论 |
|-------------------|---------------|
| 无反转幅度细化 | 🟡 逻辑更精确但实操无异 |
| 动态盈亏比调节器 | 🔴 框架不支持闭环交易记录 |
| 浮动 ATR 阈值 | 🔴 伪动态，90% 等价于固定 0.3% |
| 下午方向过滤器 | 🔴 数据证伪，53% 下午反转，过滤器会屏蔽有效窗口 |
| walk-forward 回测 | 🟢 采纳 |

### 结论

DeepSeek 的 **核心贡献**（时间作为风控变量 + 不对称持仓）被采纳，最终落地为 5 项参数修改。DeepSeek 第二轮提出的具体参数建议均未被采纳（被数据证伪或框架不支持）。

## 四、预期效果

### 方向不变

| 指标 | 当前 | 优化后 | 变化 |
|------|------|--------|------|
| 方向胜率 | 48.05% | ~48% | 不变（不修改信号层） |
| 日方向决策次数 | ~20 次 | ~12-15 次 | 早盘禁止加仓降低约 30% |

### 盈亏比改善（核心目标）

| 指标 | 当前 | 预期 | 改善逻辑 |
|------|------|------|----------|
| 持仓胜率 | 10.5% | **25-30%** | 时间止损退出浮亏持仓 |
| 平均盈利 | +1.13% | +1.0-1.5% | 止盈降级后锁利更频繁 |
| 平均亏损 | -2.14% | -0.8-1.5% | 回撤保本+时间止损限制亏损深度 |
| 盈亏比 | **0.53** | **1.0-1.5** | 主要改善项 |
| 最大回撤 | 高 | 降低 | 减少深度套牢时隙 |

### 期望收益公式

```
E(PnL) = WinRate × AvgWin - LoseRate × AvgLoss

当前：E = 10.5% × 1.13 - 89.5% × 2.14 = -1.80（每时点）
预期：E = 28%  × 1.25 - 72%  × 1.20 = -0.51（每时点，负值但缩小 72%）
```

即使方向胜率不变，盈亏比从 0.53 修复到 ≥ 1.0 后，期望收益从巨幅亏损缩小至接近中性。配合下午窗口的正确聚集，有可能实现正期望。

## 六、风险说明

1. **时间止损可能过早离场**：3 个时隙（30 分钟）无反转就减仓，如果行情在第 4 个时隙反转，会被提前止损。缓解：排除下午窗口，以及初始使用保守的 4 时隙参数。
2. **回撤 0.3% 过于敏感**：603305 日常波动中位数为 0.193%，0.3% 的波动经常因随机波动触发。缓解：强信号时段的回撤减仓比例从 50% 降为 30%。
3. **下午窗口稳定性不确定**：32.1% 胜率基于 22 个交易日，如果市场结构变化可能消失。缓解：每 5 个交易日重新计算该时段胜率，连续低于 15% 则自动降级下午加成。
